## 역할별 전문성

### 1. 사업가 (Business)

**마인드셋:**
- MVP로 빠르게 시장 검증
- 수익화보다 PMF(Product-Market Fit) 먼저
- 사용자 1,000명이 제휴보다 중요
- "만들기 전에 팔아라" (Pre-sell)

**실행 원칙:**
- 아이디어 → 3일 내 랜딩페이지
- 랜딩페이지 → 1주 내 대기자 명단 100명
- 100명 → 4주 내 MVP 출시
- MVP → 피드백 기반 주간 이터레이션

**측정 지표:**
```
Pirate Metrics (AARRR)
- Acquisition: 어디서 오는가? (트래픽 소스)
- Activation: 첫 경험은? (온보딩 완료율)
- Retention: 다시 오는가? (D1/D7/D30)
- Revenue: 돈을 내는가? (전환율)
- Referral: 추천하는가? (NPS, 바이럴)
```

### 2. 기획자 (Product)

**문서화 원칙:**
- 기획서는 개발자(자신)가 읽을 문서
- 불필요한 다이어그램 X, 코드로 표현 가능하면 코드로
- 스펙은 구체적으로, 디자인은 유연하게
- "왜?"를 항상 기록 (의사결정 히스토리)

**우선순위 프레임워크:**
```
ICE Score = Impact × Confidence × Ease
- Impact: 사용자에게 얼마나 중요한가? (1-10)
- Confidence: 성공 확신이 있는가? (1-10)
- Ease: 얼마나 쉽게 구현하는가? (1-10)
```

**기능 분류:**
```
Must Have   → MVP에 포함 (이거 없으면 제품 아님)
Should Have → Phase 2 (있으면 좋음)
Could Have  → Phase 3 (언젠가)
Won't Have  → 절대 안 함 (스코프 크립 방지)
```

**사용자 스토리 포맷:**
```
As a [사용자 유형],
I want [기능],
So that [얻는 가치].

Acceptance Criteria:
- [ ] 조건 1
- [ ] 조건 2
```

```typescript
  amount: number;
```

```typescript
// ✅ Good: Early return
function calculateNetIncome(income: number, expense: number) {
  if (income < 0) throw new Error('Invalid income');
  if (expense < 0) throw new Error('Invalid expense');
  
  return income - expense;
}

// ❌ Bad: 중첩 if
function calculateNetIncome(income: number, expense: number) {
  if (income >= 0) {
    if (expense >= 0) {
      return income - expense;
    }
  }
}
```

### 5. DevOps (배포 & 운영)

**배포 체크리스트:**
```
□ 환경변수 설정 (Vercel Dashboard)
  - NEXT_PUBLIC_SUPABASE_URL
  - NEXT_PUBLIC_SUPABASE_ANON_KEY
  - SUPABASE_SERVICE_ROLE_KEY (서버 전용)

□ Supabase 설정
  - RLS (Row Level Security) 활성화
  - 각 테이블별 정책 설정
  - Edge Functions (필요시)

□ 도메인 설정
  - Vercel에 커스텀 도메인 연결
  - SSL 자동 설정 확인
  - www → non-www 리다이렉트

□ 모니터링
  - Vercel Analytics 활성화
  - 에러 로깅 (Sentry 선택)
```

**CI/CD (Vercel 자동):**
```
main 브랜치 push → 프로덕션 배포
feature/* 브랜치 push → 프리뷰 배포

Preview URL 패턴:
https://creflow-git-feature-xxx.vercel.app
```

**환경 분리:**
```
.env.local          → 로컬 개발
.env.development    → 개발 서버
.env.production     → 프로덕션

Vercel에서 환경별로 다른 Supabase 프로젝트 연결 가능
```

**롤백:**
```bash
# Vercel CLI로 이전 배포로 롤백

# 또는 Vercel Dashboard에서 클릭 한 번
```

## 의사결정 프레임워크

### 기능 추가 요청 시

```
1. 이게 MVP에 필수인가?
   No → Phase 2로 미루기
   
2. 하루 안에 만들 수 있나?
   No → 스코프 줄이기
   
3. 사용자가 요청했나?
   No → 만들지 말기
```

### 기술 선택 시

```
1. 이미 아는 기술인가?
   Yes → 사용
   No → 학습 비용 vs 이득 비교
   
2. 무료 티어 있나?
   No → 대안 찾기
   
3. 나중에 교체 가능한가?
   No → 신중하게 결정
```

### 버그 vs 기능 우선순위

```
Critical Bug (데이터 손실, 보안) → 즉시 수정
Major Bug (핵심 기능 불가) → 당일 수정
Minor Bug (불편함) → 백로그
Feature Request → 백로그 + 투표
```
